Frequently Asked Questions

What is the difference between var, let, and const?
-var: Older way to declare variables, can be changed and has function scope.
- let: More modern, can be changed, and has block scope.
- const: Used for constants, cannot be changed after being assigned, and has block scope.
What is the difference between map(), forEach(), and filter()
- **map()**: Creates a new array by applying a function to each element of the original array.
- **forEach()**: Loops through each element of the array but doesn't return anything.
- **filter()**: Creates a new array with only elements that pass a condition.
What are arrow functions, and how are they different from regular functions?
- **Arrow functions**: Shorter syntax for writing functions, using `=>`, and they don't have their own `this` context.
- **Difference**: Regular functions have their own `this` context, while arrow functions inherit `this` from the surrounding scope.
How do JavaScript Promises work?
JavaScript *Promises* are used to handle asynchronous operations.
They can be in one of three states:
1. Pending: The operation is still ongoing.
2. Resolved (Fulfilled): The operation is successful, and the promise is completed.
3. Rejected: The operation failed.
You use `.then()` to handle a successful outcome and `.catch()` to handle errors.
How does closure work in JavaScript?
A **closure** is when a function remembers and can use variables from its outer function, even after it’s done running.
It lets inner functions keep access to outer function's variables.
This is useful for creating private variables and functions.
